# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Email:** dradamopierce@gmail.com

Please include:
- A description of the vulnerability
- - Steps to reproduce the issue
  - - Potential impact
   
    - I will acknowledge receipt within 48 hours and aim to provide a fix or mitigation plan within 7 days.
   
    - ## Supported Versions
   
    - | Version | Supported |
    - | ------- | --------- |
    - | Latest  | Yes       |
   
    - ## Security Practices
   
    - - Environment variables are used for all secrets and credentials
      - - `.env` files are excluded from version control via `.gitignore`
        - - Dependencies are monitored for known vulnerabilities
